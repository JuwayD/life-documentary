# 2026-06-10 Phase 8: 事件时间线筛选

## 本次目标

继续 Phase 8 信息展示优化,为事件时间线增加按类型筛选能力,减少复盘时的信息噪音。

## 实现内容

- 事件时间线面板新增筛选按钮:
  - 全部
  - 剧情
  - 社交
  - 战斗
  - 交易
  - 其他
- 筛选按钮展示最近事件中的类型数量,无内容的类型自动置灰。
- 筛选状态在当前页面内保留;若新状态中该类型不存在,自动回到“全部”。
- 时间线从最近 5 条扩展为最近 8 条,让筛选后仍有足够上下文。
- 新增筛选按钮样式。
- 补充 Playwright E2E 测试:
  - 验证筛选入口可见。
  - 验证社交/剧情筛选只展示对应事件类型。
- 新增 ADR-0031。

## 设计原则

- 只复用现有 `/api/state` 的 `events`,不新增后端接口、状态字段或工具。
- 前端只做展示分类与筛选,不影响事件真实记录和 GM Agent 判断。
- 保持最小垂直切片:JS 渲染、CSS 样式、E2E 验收与文档同步。
