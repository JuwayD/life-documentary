# 2026-06-09 Phase 8: 局势摘要面板

## 本次目标

继续 Phase 8 信息展示优化,选择上一轮会话记录中最靠前且可独立验收的“重要信息摘要”小切片。

## 实现内容

- Web 右栏新增“局势摘要”面板,放在时间面板之后。
- 摘要展示当前位置、在场人数、线索总数、最高压力钟、当前行动角色和最新事件。
- 摘要只从已有 `/api/state` 快照派生,不新增 API 或规则判断。
- 摘要面板接入已有折叠/展开与 `localStorage` 偏好机制。
- 补充 Playwright E2E 测试,验证初始摘要和线索/压力变化后的摘要更新。
- 新增 ADR-0026。

## 设计原则

- 只优化浏览器端信息层级,不改变世界状态、工具层、GM Agent 或审计日志。
- 摘要展示“事实”,不替玩家或 AI 判断下一步。
- 保持最小垂直切片:一个入口、一个渲染函数、样式和端到端测试。
