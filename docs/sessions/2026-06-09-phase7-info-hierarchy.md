# 2026-06-09 Phase 7: 事件/线索信息层级优化

## 本次目标

继续 Phase 7 玩法打磨,选择 roadmap 中最靠前的可独立验收小切片:优化事件与线索信息层级,让玩家更容易快速理解局势。

## 实现内容

- 时间线事件新增类型标签与“最新”标签。
- 类型标签仅根据已有 `type` / `action_type` / `summary` 做展示分类,不影响 GM 决策。
- 线索面板新增总览行:线索总数与线程数。
- 线索线程排序:主线优先,支线按线索数量与名称排序。
- 线程内线索按新近优先显示,最新线索标记“新近”。
- 补充 Playwright E2E 覆盖时间线标签与线索层级展示。
- 新增 ADR-0020。

## 设计原则

- 不新增后端 API,继续复用 `/api/state`。
- 不做剧情/线索重要性判断;代码只做 UI 层级与事实展示。
- 保持最小垂直切片,可由浏览器测试独立验收。
