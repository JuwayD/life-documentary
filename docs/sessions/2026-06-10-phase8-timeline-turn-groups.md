# 2026-06-10 Phase 8: 时间线回合分组

## 本次目标

继续 Phase 8 信息展示优化。在已有时间线筛选、摘要与最近变化面板基础上，选择最小且可独立验收的后续切片：时间线回合分组。

## 实现内容

- 前端事件时间线列表新增分组标题：
  - 有 `turn` 的事件显示为“回合 N”。
  - 无 `turn` 但有 `timestamp` 的事件按时间戳分组。
  - 两者都缺失时归入“未分组事件”。
- 分组应用于当前筛选后的事件列表；切换剧情/社交/战斗等筛选时，分组随可见事件重新计算。
- 新增 `.timeline-group` 样式，以虚线分隔符展示，不改变事件卡片本身结构。
- 补充 Playwright E2E 测试：
  - 直接渲染带不同 `turn` 的事件快照。
  - 验证时间线显示两个回合分组。
  - 切换社交筛选后只保留对应回合分组。
- 新增 ADR-0058。

## 设计原则

- 只改前端展示层，不新增 AI 决策逻辑。
- 保持历史事件兼容：缺少 `turn` 时自动降级为时间戳或未分组。
- 分组标题只在相邻分组变化时出现，避免重复噪音。
