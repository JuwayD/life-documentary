# 2026-06-09 Phase 8: 右栏信息导航

## 本次目标

继续 Phase 8 信息展示优化,在不改后端和规则判断的前提下,减少右栏信息密集后的查找成本。

## 实现内容

- Web 右栏顶部新增粘性“信息导航”。
- 导航分为局势、行动、剧情、复盘四个入口,分别跳转到对应面板。
- 点击导航时自动展开目标面板,并同步已有 `localStorage` 折叠偏好。
- 增加样式使导航在右栏滚动时保持可见。
- 补充 Playwright E2E 测试,验证导航可重新展开已折叠的剧情面板。
- 新增 ADR-0027。

## 设计原则

- 只优化展示层导航,不新增 API、不改变 World、工具层、GM Agent 或审计日志。
- 导航入口选择现有信息结构中最常用的四类分组,避免继续增加信息噪音。
- 保持最小垂直切片:一个 DOM 入口、一个初始化函数、样式、测试和文档。
