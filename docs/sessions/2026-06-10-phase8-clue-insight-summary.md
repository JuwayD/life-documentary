# 2026-06-10 Phase 8: 线索摘要条

## 本次目标

继续 Phase 8 信息展示优化。在已有线索记录面板基础上，选择最小且可独立验收的后续切片：线索摘要条。

## 实现内容

- 前端线索记录面板在总览下方新增摘要区域：
  - 显示最新记录的线索。
  - 显示当前线索数量最多的活跃线程。
- 摘要复用已有 `story_progress` 与 `story_seeds` 数据，不新增世界状态、工具或后端接口。
- 新增 CSS 样式，使摘要以紧凑双列信息块展示，位于总数概览与线程列表之间。
- 补充 Playwright E2E 断言：触发线索记录后验证摘要显示“最新线索”、最新线索文本与“活跃线程”。
- 新增 ADR-0042。

## 设计原则

- 只优化信息展示，不改变线索记录与排序规则。
- 不把调查判断写入代码，仍由 GM Agent 根据世界状态与玩家输入决策。
- 摘要提供快速定位，详细内容仍由原线索分组承载。
