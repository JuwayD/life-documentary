# 会话日志: 2026-06-06 Phase 1 — Web UI

## 触发

用户提问:Python 怎么测 UI?Web 有内置浏览器方便调试。

## 决策

把原本规划在 Phase 3 的 Web 渲染提前到 Phase 1。理由:
- Pygame UI 测试成本太高
- Playwright + 浏览器在可测性、可分享性、远程调试上是降维打击
- 现有 Python 代码零迁移,只在外面加 FastAPI 薄层

## 文档产出

- `docs/decisions/0005-web-ui-layer.md` — Web 架构决策(ADR)

## 代码产出

### 后端 (Python)
- `src/mingrpg/web/server.py` — FastAPI app factory + 5 个端点
  - GET `/` 静态 HTML
  - GET `/api/state` 当前快照
  - POST `/api/turn` 一回合
  - GET `/api/audit?limit=N` 决策日志
  - POST `/api/reset` 重置世界

### 前端 (零构建工具)
- `src/mingrpg/web/static/index.html` — 单页 SPA
- `src/mingrpg/web/static/style.css` — 明代纸质风(朱砂+石青)
- `src/mingrpg/web/static/app.js` — Vanilla JS 业务逻辑

### 测试
- `tests/web/test_server.py` — 7 个 FastAPI 单元测试 (TestClient)
- `tests/web/test_e2e_browser.py` — 7 个 Playwright 浏览器测试
  - 用 FakeAgent 避开 LLM 调用,保证快速 + 确定性
  - 测了开场、状态面板、NPC 列表、输入提交、状态变更、审计弹窗、重置

### 工具
- `run_web.sh` — 启动 + 自动打开浏览器
- `pyproject.toml` 新增依赖:fastapi / uvicorn / httpx / pytest-playwright / pytest-asyncio

## 踩坑

1. **SQLite 多线程异常** — FastAPI sync endpoints 跑在 threadpool,SQLite 默认拒绝跨线程。改 `check_same_thread=False`,因为游戏天然单玩家无并发写。

2. **FakeAgent 不能直接 mutate world** — 必须也走 `audit.start_turn/end_turn`,否则审计端点取不到。这反向验证了"GMAgent 协议"的清晰度:audit 是契约的一部分。

## 测试总览

| 类型 | 数量 | 耗时 |
|---|---|---|
| 单元测试 | 55 | 0.3s |
| Web 后端 (TestClient) | 7 | 0.5s |
| 浏览器 E2E (Playwright) | 7 | 4s |
| LLM 集成 (skip 默认) | 4 | 4 分钟 (需 MINGRPG_RUN_LIVE=1) |

**69 + 4 = 73 个测试,全过。**

## 关键观察

**Playwright 真香**: 7 个浏览器测试 4 秒跑完,headless chromium 零配置启动。Vanilla JS 完全够用,不需要 React/Vue。

**纸质风格 UI**: 朱砂红 (#8b3a2f) + 古纸黄 (#f4ecd8) + 衬线字体,直接出明代感。决策日志面板可在浏览器里点开,看 AI 调用了哪些工具、引了哪些法条 — **这就是"AI 友好 + 人类可审核"在 UI 层的体现**。
