# 2026-06-11 Phase 9 调试快照删除确认

## 本次目标

继续打磨 Phase 9 开发者控制台的调试快照管理体验,为破坏性删除操作补上最小安全确认切片。

## 完成内容

- 单个测试快照删除前增加浏览器确认。
- 批量测试快照删除前增加浏览器确认,提示即将删除的快照数量。
- 取消确认时不调用后端删除 API,快照仍保留在列表中。
- 确认后沿用既有删除流程,并刷新控制台状态。
- 补充 Playwright E2E 覆盖单个删除取消/确认与批量删除确认文案。

## 设计约束

- 不修改后端 API 与快照文件格式。
- 不改变归档/导出/导入等非破坏性操作。
- 只在前端阻止误触,保留脚本直接调用调试 API 的能力。

## 测试

- `tests/web/test_e2e_browser.py` 覆盖单个快照删除确认与取消。
- `tests/web/test_e2e_browser.py` 覆盖批量删除确认提示包含所选数量。
