# 开发路线

## Phase 0: 端到端文字闭环 (当前阶段)

**目标**: 跑通 "玩家输入 → GM Agent 决策 → 工具调用 → 世界变更 → 叙述输出" 的最小循环。

### 验收标准
- [ ] 玩家可在 CLI 输入任意自然语言
- [ ] GM Agent 能调用至少 10 个工具完成决策
- [ ] 不合理行为被 AI 合理驳回并引用依据
- [ ] 全过程有可审计日志
- [ ] 所有代码通过 TDD 编写,覆盖率 > 80%

### 任务拆分
1. **项目骨架** — 目录结构、pyproject.toml、pytest 配置
2. **ADR 撰写** — 4 条核心架构决策记录
3. **State Layer** — SQLite schema、World 抽象 (TDD)
4. **Read Tools** — get_entity / get_location / list_nearby / query_laws (TDD)
5. **Write Tools** — set_attribute / move_entity / add_status / log_event (TDD)
6. **Util Tools** — roll_dice / calc_distance (TDD)
7. **Law 数据** — 5 条明律 YAML
8. **GM Agent** — 系统提示词 + Claude Tool Use 集成
9. **Audit Layer** — JSONL 日志 + 简易 review CLI
10. **端到端 Demo** — "扬州府衙大堂" 场景,验证打知府场景

### 出口条件
能在终端运行 `python -m mingrpg` 进入游戏,输入 "我冲上去给知府一拳" 得到合理回应。
