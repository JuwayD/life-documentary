# ADR-0081: Phase 9 调试快照批量清理

## 状态

Accepted

## 背景

开发者控制台已支持调试快照保存、加载、筛选、排序、单个删除和列表摘要。随着回归验证快照增多，逐个删除临时快照会拖慢调试整理流程，尤其是按名称或备注筛选出一组临时盘面后。

## 决策

为测试场景快照增加批量选择与删除能力:

1. 新增 `POST /api/debug/test-snapshots/bulk-delete`，接收 `snapshot_ids`。
2. 后端对快照 id 复用既有路径校验，去重后逐个删除，并返回已删除元数据与删除后的摘要。
3. 开发者控制台快照列表增加“选择当前列表”“已选数量”“删除所选”“清空选择”。
4. 批量删除只影响调试快照文件，不改变当前世界状态、审计日志或正式存档。

## 后果

- 调试者可先筛选临时快照，再一键清理当前筛选结果。
- 删除仍由用户在控制台显式触发，不自动清理。
- 如果选择中包含不存在的快照，接口返回 404，避免静默忽略误选或陈旧状态。
