# ADR-0082: Phase 9 调试快照批量导出

## 状态

Accepted

## 背景

开发者控制台已支持单个调试快照导出、索引导出和批量删除。实际调试时常需要把一组相关复现盘面一起分享或归档，逐个导出会增加操作成本，而索引导出不包含完整存档内容，不能直接复现。

## 决策

为测试场景快照增加批量导出能力:

1. 新增 `POST /api/debug/test-snapshots/bulk-export`，接收 `snapshot_ids`。
2. 后端复用快照 id 去重与路径校验，返回 `mingrpg.test-snapshot-bundle` JSON。
3. 包内每个快照保留完整 `save`，并标记为 `mingrpg.test-snapshot`，便于后续单个恢复或人工拆分。
4. 开发者控制台复用已选快照状态，新增“导出所选”按钮，下载本地 JSON 包。
5. 批量导出只读快照文件，不改变当前世界状态，也不删除调试快照。

## 后果

- 调试者可先筛选一组快照，再一键导出完整复现包。
- 导出包可能较大，但仅由用户显式触发并保存到本地。
- 当前导入仍保持单个快照导入，批量包导入留作后续可独立切片。
