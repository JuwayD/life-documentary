# ADR-0114: NPC 对话系统

## 状态

已接受

## 背景

NPC 交互完全由 LLM 自由生成,缺乏结构化对话素材支撑。NPC 的问候、话题、告别等台词没有持久化存储,导致:
- 同一 NPC 在不同回合中语气可能不一致
- 无法根据态度值动态调整可用对话
- GM 缺少可参考的台词库,每次需要凭空创造

## 决策

引入 NPC 对话系统:

1. **对话素材**: NPC `attributes.dialogue_lines` 存储结构化台词库,包含:
   - `greetings`: 按态度范围过滤的问候语
   - `topics`: 按 `unlock_attitude` 解锁的话题,每个话题有多条按态度过滤的台词
   - `farewells`: 告别语
   - `special`: 特殊触发台词(首次见面/高态度/低态度)

2. **读取工具**: `get_npc_dialogue(npc_id, player_id)` 根据 NPC 对玩家的态度值过滤并返回可用对话选项

3. **写入工具**: `record_dialogue(npc_id, player_id, topic, ...)` 记录对话交互:
   - 添加 NPC 记忆
   - 可选调整态度值
   - 记录世界事件

4. **态度系统**: 复用现有 `attributes.attitude[player_id]` 字段,范围 -100~+100

5. **前端面板**: 右栏"对话"面板展示附近 NPC 的对话选项、态度状态和特殊台词

## 不做的事

- 不实现对话树或分支选择(保持 LLM 自由叙述)
- 不自动触发对话(由 GM 根据情境判断)
- 不强制使用对话工具(简单闲聊 GM 可自行处理)

## 后续

- 为更多 NPC 补充对话素材
- 考虑对话历史追踪面板
- 态度变化可视化趋势
