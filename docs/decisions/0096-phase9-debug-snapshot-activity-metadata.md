# ADR-0096: Phase 9 调试快照活动时间元数据

## 状态

Accepted

## 背景

开发者控制台已经支持测试快照的保存、加载、导出、导入、批量管理、健康校验与快速筛选。快照数量增加后,调试者需要判断某个快照是否最近被加载复现过、是否已经导出分享过,但此前列表只展示创建/更新时间。

## 决策

为测试快照补充轻量活动时间元数据:

- 加载本地测试快照时写入 `last_loaded_at`。
- 导出单个测试快照时写入 `last_exported_at`,并在导出文件中保留该字段。
- 单文件导入与快照包导入返回/保存 `last_imported_at`。
- 快照列表与详情返回这些字段,前端在快照行展示“加载/导出/导入”最近活动时间。

该能力只记录调试操作痕迹,不改变快照内 `save.world` 内容,也不新增游戏规则判断。

## 后果

- 调试者可以从快照列表直接判断最近复现与分享状态。
- 导出后列表会刷新,让 `last_exported_at` 立即可见。
- 元数据属于调试辅助信息,不会影响世界状态导入导出语义。
