# ADR-0030: Phase 8 关系动态面板

## 状态

Accepted

## 背景

Phase 8 已通过局势摘要、待处理重点和右栏导航降低信息查找成本,但 NPC 相关信息仍主要散落在“附近的人”“顾问”“事件时间线”中。玩家在与 NPC 互动后,不容易快速判断在场人物的身份、态度线索和最近互动记忆。需要补一个信息展示切片,但不能让前端替 GM Agent 判定好感、敌意或剧情结果。

## 决策

- 在“附近的人”之后增加“关系动态”面板。
- 面板只读取现有 `/api/state` 中在场 NPC 的字段:
  - `attributes.occupation` / `rank` 展示身份。
  - `attributes.personality` 展示态度线索。
  - `attributes.memories` 展示最近一条互动记忆、记忆数量和重要度。
- 不新增 API、World 字段、工具或 GM Agent 规则。
- 没有记忆时显示“尚无互动记忆”,避免编造关系状态。

## 后果

- 玩家可在当前场景快速复盘“谁在场、是什么身份、刚刚记住了什么”。
- 面板保持展示层聚合,不引入前端规则判断或数值化好感系统。
- 若后续 NPC 记忆结构扩展,该面板只需同步展示字段,不影响状态层。
