# ADR-0052: Phase 8 线索事件联动

## 状态

Accepted

## 背景

Phase 8 已分别优化线索记录与事件时间线。玩家在线索面板看到“新近线索”后，若想回看该线索是在什么上下文中出现的，需要手动滚动到事件时间线并识别对应事件。

## 决策

- 在线索卡片上增加“查看事件”入口。
- 前端基于现有 `state.events` 中的 `record_clue` 事件，将线索与对应事件建立展示层关联。
- 点击入口时重置事件时间线筛选为“全部”，展开并聚焦事件时间线面板，高亮对应事件卡片。
- 不新增后端接口，不改变世界状态、工具调用、审计日志或 GM 决策。

## 后果

- 玩家可从线索直接回到事件上下文，降低跨面板查找成本。
- 该功能保持信息展示增强性质，可独立验收。
- 如果未来事件快照限制导致较早线索事件不在 `state.events` 中，线索仍正常展示，只是不显示跳转入口。
