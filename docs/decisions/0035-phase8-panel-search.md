# ADR-0035: Phase 8 右栏面板搜索筛选

## 状态

Accepted

## 背景

Phase 8 持续优化信息展示。右栏已经支持折叠、导航、摘要、重点、关系动态、时间线筛选、信息密度切换与移动端抽屉入口；但随着面板数量增加，玩家在寻找特定信息（如线索、顾问、压力钟、NPC 名称）时仍需要滚动浏览多个分组。

## 决策

- 在右栏信息密度切换下方加入搜索框。
- 搜索框按当前已渲染的面板标题与内容进行本地筛选：
  - 输入关键词后，仅显示 `textContent` 匹配的面板。
  - 无匹配时显示“无匹配面板”。
  - 清空输入后恢复全部面板。
- 状态刷新或面板重渲染后，保留当前搜索词并重新应用筛选。
- 不新增后端接口、世界状态、工具层或 GM Agent 规则。

## 后果

- 玩家可以直接定位右栏中的线索、压力、顾问、关系或复盘信息。
- 实现完全停留在前端展示层，风险低，且可通过 E2E 独立验收。
- 搜索基于当前 DOM 文本，后续若面板内容进一步结构化，可再扩展为高亮匹配或分组索引。
