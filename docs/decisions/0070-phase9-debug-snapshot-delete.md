# ADR-0070: 调试快照删除

## 背景

开发者控制台已有测试快照保存、加载、差异预览和元数据编辑能力。随着调试迭代增多，临时快照会堆积，影响定位常用测试盘面。

## 决策

为测试快照增加删除端点和前端删除入口，让开发者可以清理无用快照。删除仅作用于调试快照 JSON 文件，不影响当前世界状态、审计日志或其他存档。

## 方案

- **后端**：新增 `DELETE /api/debug/test-snapshots/{snapshot_id}`，读取快照元数据后删除对应 JSON 文件
- **前端**：快照操作区新增“删除”按钮，删除后刷新开发者控制台并清理该快照差异缓存
- **测试**：覆盖删除成功后列表、diff、load 均不可再访问，以及未知快照 404

## 未选择的方案

- **批量删除**：容易误删多个调试盘面，先保留单快照删除的可控操作
- **软删除**：调试快照是可再生成的本地文件，软删除会增加列表过滤与清理复杂度

## 影响

- 后端新增 1 个 DELETE 端点，无数据迁移
- 前端调试快照区域新增一个危险操作样式按钮
- 当前世界状态不随删除改变
