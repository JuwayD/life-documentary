# ADR-0028: Phase 8 右栏导航摘要徽标

## 状态

Accepted

## 背景

Phase 8 已提供右栏信息导航,但导航入口本身只负责跳转。玩家仍需要点进或滚动到面板后,才能判断当前是否有新线索、可行动建议、在场人数或复盘事件量。需要继续优化信息展示密度,但不能把规则判断写入前端或后端。

## 决策

- 在右栏信息导航的 4 个入口下增加轻量摘要徽标。
- 徽标直接从 `/api/state` 已有快照计算:
  - 局势:当前位置在场 NPC 数。
  - 行动:当前可展示行动建议数量。
  - 剧情:已记录线索数量。
  - 复盘:近期事件数量。
- 压力钟达到危险线 80% 时,局势徽标使用强调样式。
- 不新增 API、World 字段、工具或 GM Agent 规则。

## 后果

- 玩家无需展开面板即可知道关键分组的信息量。
- 导航继续只是展示层聚合,所有数值来自现有快照。
- 行动建议数量需要与前端建议生成逻辑保持一致;若未来建议规则调整,同步更新计数函数即可。
