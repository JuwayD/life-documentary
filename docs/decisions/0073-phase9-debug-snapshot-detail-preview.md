# ADR-0073: 调试快照详情预览

## 背景

开发者控制台已支持保存、加载、差异预览、编辑和删除测试快照，但加载前只能看到列表中的计数与备注。调试多个相似盘面时，仍需要通过差异或直接加载来确认快照内容，容易打断当前调试上下文。

## 决策

为测试快照增加只读详情预览：开发者可在加载前查看玩家位置、实体/地点/事件/Flag 数量、Flag 键列表和最近事件摘要。

## 方案

- **后端**：新增 `GET /api/debug/test-snapshots/{id}`，读取快照文件并返回轻量摘要，不导出完整存档正文
- **前端**：在每条快照记录增加“详情”按钮，点击后展开/收起详情预览
- **展示内容**：玩家、当前位置、计数、Flag 键和最近 5 条事件
- **状态管理**：删除、加载或元数据更新快照时清理对应详情缓存
- **测试**：单元测试覆盖详情端点与 404；E2E 覆盖控制台详情按钮和内容展示

## 未选择的方案

- **直接展示完整 save JSON**：信息完整但噪音过大，且与调试包导出能力重复
- **仅复用差异预览**：差异依赖当前世界，不能回答“这个快照自身是什么盘面”

## 影响

- 无数据迁移
- 无破坏性 API 变更
- 新端点只读，不改变世界或快照内容
