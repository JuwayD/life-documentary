# ADR-0091: Phase 9 调试快照健康校验

## 状态

Accepted

## 背景

Phase 9 开发者控制台已支持保存、导入、导出、复制、置顶、归档和批量管理调试快照。随着本地快照数量增加,调试目录可能出现手工编辑、旧格式导入、重复命名或文件损坏导致的不可加载快照。直接加载时才发现问题会打断调试流程。

## 决策

新增只读的调试快照健康校验能力：

- 后端提供 `GET /api/debug/test-snapshots/health`,扫描本地测试快照目录。
- 校验项覆盖 JSON 损坏、缺失 id/name、重复名称、非测试快照/存档格式、缺少玩家实体、缺少地点数据。
- 返回总数、正常数、问题数和逐快照问题列表。
- 前端开发者控制台新增“校验快照”按钮,展示校验通过或问题报告。

该能力只读取和报告快照状态,不自动修复、不删除、不加载快照,避免改变调试资产或当前世界。

## 后果

- 调试者可以在整理快照前快速发现坏档、重复命名和结构缺口。
- 校验报告可作为后续手工归档、删除或重新导入的依据。
- 后端新增只读 API,不改变快照文件格式、存档导入流程或游戏规则判断。
