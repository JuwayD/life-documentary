# ADR-0088: Phase 9 调试快照归档

## 状态

Accepted

## 背景

Phase 9 开发者控制台已支持测试快照保存、筛选、排序、标签、复制、置顶与批量导入导出。随着临时验证快照累积，删除会丢失复盘材料，但全部显示又会干扰常用快照列表。

## 决策

为测试快照新增归档能力：

- 快照元数据新增 `archived` 布尔字段，默认 `false`。
- `PATCH /api/debug/test-snapshots/{snapshot_id}` 支持更新 `archived`，并刷新 `updated_at`。
- 列表、详情、索引导出、单个导出、复制、批量导入预览与导入均保留 `archived` 元数据。
- 开发者控制台默认隐藏已归档快照，并显示“已归档”计数。
- 控制台提供“显示归档”开关；已归档快照显示“归档”徽标，并可一键取消归档。

归档只影响调试快照元数据和控制台展示，不删除快照文件，不读取、不修改世界状态，也不参与游戏规则判断。

## 后果

- 临时快照可以从默认列表中降噪隐藏，同时保留后续复盘和导出能力。
- 归档状态随导出、复制、批量包预览和导入保留，便于跨环境传递快照整理结果。
- 旧快照没有 `archived` 字段时按未归档处理，兼容既有本地快照文件。
