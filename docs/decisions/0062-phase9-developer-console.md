# ADR-0062: Phase 9 开发者控制台最小切片

## 状态

已接受

## 背景

Phase 9 的目标是提供测试与调试工具。出生设置系统已完成后，roadmap 中最靠前的下一项是“开发者控制台”，用于在游玩和测试中快速查看世界状态、工具调用日志与性能指标。

现有系统已有：
- `/api/state`：世界快照。
- `/api/audit`：审计回合记录。
- 前端审计日志弹层：面向决策追踪。

但测试者仍需要在多个入口之间切换，才能同时判断“当前世界状态是什么、最近调用过哪些工具、调试规模如何”。

## 决策

实现一个只读的开发者控制台最小垂直切片：

- 新增 `GET /api/debug/console?limit=` 聚合调试接口。
- 返回三类信息：
  - `world`：实体数、地点数、事件窗口、flag 数、时间、玩家、当前位置。
  - `audit`：审计总回合数、最近工具调用。
  - `performance`：审计日志字节数、快照事件窗口大小。
- 前端顶部新增“开发者控制台”入口。
- 弹层内分区展示：世界状态查看器、工具调用日志、性能监控。

## 约束

- 控制台只读，不提供修改世界、删除日志或执行工具能力。
- 不新增游戏规则判断；后端只聚合已有状态与审计数据。
- 工具调用日志只展示近期调用，避免把完整审计日志一次性塞到 UI。
- 继续保留既有“查看日志”入口；开发者控制台是调试概览，不取代审计详情。

## 后果

### 正面

- 测试者能在一个入口快速确认当前世界、最近工具调用与调试规模。
- 为后续更细的世界状态查看器、工具日志筛选、性能监控面板预留 UI 位置。
- 不影响玩家主循环，不触碰 GM Agent 决策职责。

### 负面

- 顶部工具栏按钮继续增加，后续可能需要管理/调试菜单归纳。
- 当前性能指标较轻量，只覆盖审计大小和前端渲染统计，尚不是完整 profiling。
