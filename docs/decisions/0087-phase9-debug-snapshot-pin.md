# ADR-0087: Phase 9 调试快照置顶

## 状态

Accepted

## 背景

Phase 9 开发者控制台已支持测试快照保存、筛选、排序、标签、复制与批量导入导出。随着回归盘面增多，常用快照容易被新建或导入的临时快照挤到列表后方，调试时需要反复搜索。

## 决策

为测试快照新增置顶能力：

- 快照元数据新增 `pinned` 布尔字段，默认 `false`。
- `PATCH /api/debug/test-snapshots/{snapshot_id}` 支持更新 `pinned`，并刷新 `updated_at`。
- 列表、详情、索引导出、单个导出、复制、批量导入预览与导入均保留 `pinned` 元数据。
- 测试快照列表默认将置顶项排在非置顶项之前；前端排序也保持置顶优先。
- 开发者控制台每个快照行新增“置顶 / 取消置顶”按钮，并显示“置顶”徽标。

置顶只影响调试快照元数据和控制台展示顺序，不读取、不修改世界状态，也不参与游戏规则判断。

## 后果

- 常用回归盘面可以固定在列表顶部，降低查找成本。
- 置顶状态随导出、复制、批量包预览和导入保留，便于跨环境复用常用快照集合。
- 旧快照没有 `pinned` 字段时按未置顶处理，兼容既有本地快照文件。
