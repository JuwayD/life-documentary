# ADR-0083: Phase 9 调试快照批量导入

## 状态

Accepted

## 背景

开发者控制台已支持按筛选结果批量导出完整测试快照包。为了让导出的复现包能在另一份本地环境中快速恢复为可浏览、可加载的调试快照列表，需要配套的批量导入能力。

## 决策

为测试场景快照增加批量导入能力:

1. 新增 `POST /api/debug/test-snapshots/bulk-import`，接收 `mingrpg.test-snapshot-bundle`。
2. 后端先校验包格式、版本与每个 `mingrpg.test-snapshot` 的完整 `save` 结构，再写入任何快照，避免半导入。
3. 导入时为每个快照生成新的本地 id，保留原名称、备注和完整世界存档。
4. 批量导入只追加到调试快照列表，不自动加载世界状态，避免意外覆盖当前调试盘面。
5. 开发者控制台新增“导入快照包”按钮，导入后刷新快照列表并清空选择状态。

## 后果

- 调试者可将一组复现盘面从一台机器导出，并在另一台机器一键导入为本地快照列表。
- 新 id 避免覆盖现有同名或同源快照。
- 若需要恢复某个导入快照，仍通过已有“加载”确认流程单独执行。
