# ADR-0072: 调试工具调用 JSON 复制

## 背景

开发者控制台已展示工具调用输入与输出，但排查问题或分享调试片段时仍需要手动选中文本复制，容易漏选或破坏 JSON 格式。

## 决策

在工具调用日志每条记录的输入与输出区域增加“复制输入 / 复制输出”按钮，一键复制对应 JSON。

## 方案

- **前端**：渲染工具调用时将 input/output JSON 放入复制按钮的 data 属性
- **交互**：点击后通过统一复制函数写入剪贴板，优先使用 `navigator.clipboard.writeText`，必要时回退到隐藏 textarea，并将按钮文案改为“已复制输入/输出”
- **样式**：输入/输出标签行右侧放置轻量复制按钮
- **后端**：无改动，复用现有 `/api/debug/console` 数据
- **测试**：E2E 覆盖复制按钮可见与点击后反馈

## 未选择的方案

- **复制整条工具调用**：数据更完整，但常见排查需要分别分享输入或输出，颗粒度过粗
- **下载单条 JSON 文件**：适合归档，但对即时调试过重

## 影响

- 无数据迁移
- 无 API 破坏性变更
- 依赖浏览器剪贴板能力；测试仅验证按钮与点击反馈
