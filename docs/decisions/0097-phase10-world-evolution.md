# ADR-0097: Phase 10 世界演化系统

## 状态

Accepted

## 背景

当前世界的运作方式是"玩家动，世界才动"。NPC 只在被交互时才"活"，时间只在玩家行动时才推进。玩家希望世界像真实世界一样持续运转——即使玩家什么都不做，NPC 有自己的生活，事件在发生，世界在变化。

项目的核心原则是**代码不做决策**：代码是手脚，AI 是大脑。任何 NPC 行为、世界事件的决策都必须由 AI 做出，代码只负责执行和记录。

## 决策

引入**演化注册表**机制，由 AI 管理世界的持续变化。

### 演化注册表

在世界状态中维护一个 `evolution_registry`（flag），记录哪些实体需要被演化：

```json
{
  "evolution_registry": [
    {
      "entity_id": "beggar",
      "frequency": "every_2_turns",
      "last_evolved_turn": 5,
      "reason": "玩家常在街市活动，乞丐是常驻 NPC"
    }
  ]
}
```

### 核心循环

每个玩家回合，GM Agent 在一次 LLM 调用中同时处理：

1. 读取演化注册表，判断哪些条目到期
2. 对到期条目做出演化决策（移动、改变状态、产生事件等）
3. 判断注册表是否需要调整（加入/移除/改频率）
4. 处理玩家行为
5. 返回叙述，包含"你不在时世界发生了什么"

### 频率语义

频率由 AI 决定，使用语义化表达：

| 频率 | 含义 |
|---|---|
| `every_turn` | 每回合都演化（玩家就在眼前） |
| `every_2_turns` | 隔一回合 |
| `every_5_turns` | 低频关注 |
| `dormant` | 暂停演化，等被激活 |

### 频率调整规则

AI 根据以下因素动态调整频率：
- **空间距离**：玩家附近的实体频率升高，远处的降低
- **叙事关联**：与当前线索、压力钟、主线相关的实体频率升高
- **时间衰减**：长时间无交互的实体频率降低
- **事件驱动**：重大事件涉及的实体临时升高

### 什么会上演化列表

由 AI 决定，参考因素：
- 玩家附近的 NPC
- 和当前线索/压力钟相关的实体
- 有独立动机的重要 NPC
- 环境因素（天气、物价等）
- 玩家触发但未完成的事件的后续

### 新增工具

- `register_evolution(entity_id, frequency, reason)` — 加入演化列表
- `update_evolution(entity_id, frequency, reason)` — 调整频率
- `remove_evolution(entity_id)` — 移除
- `list_evolutions()` — 查看当前注册表

### 成本控制

- 演化决策和玩家行动决策合并在一次 LLM 调用中，不额外增加 API 调用
- 远处无关的条目频率低，消耗极少 token
- `dormant` 状态的条目不消耗任何 token

## 后果

- 世界在玩家不行动时依然在变化，增强沉浸感
- 所有决策权仍在 AI，遵守"代码不做决策"原则
- 成本可控：通过频率机制避免每 tick 都推演所有 NPC
- 演化列表本身也是 AI 的决策，可随剧情动态调整
- GM 叙述可自然融入演化结果，不需要单独的叙述流程
