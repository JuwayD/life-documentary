# ADR-0066: Phase 9 调试包导出

## 状态

已接受

## 背景

Phase 9 的开发者控制台已经可以查看世界状态、实体/flag、工具调用、性能指标、测试快照和预设测试用例。但当测试者需要向他人复现问题或保存当前调试证据时，只能手动截图或分别复制多个面板信息，容易遗漏筛选条件和工具调用上下文。

## 决策

在开发者控制台中加入“导出调试包”的最小切片：

- 新增 `GET /api/debug/export`，复用开发者控制台聚合数据结构。
- 导出内容包含当前世界摘要、实体/flag、筛选后的工具调用、工具筛选条件与性能指标。
- 导出数据增加 `format: mingrpg.debug-bundle`、`version: 1`、`exported_at`，便于识别来源。
- 前端开发者控制台新增“导出调试包”按钮，按当前工具名/内容筛选条件下载 JSON。
- 导出是只读调试能力，不改变世界状态、审计日志或测试快照。

## 约束

- 不引入外部上传、分享链接或账号系统，避免把本地调试数据发布到第三方。
- 继续限定在 `/api/debug/*` 调试接口下，不影响正式游玩接口。
- 复用现有筛选逻辑，避免另建一套导出查询规则。

## 后果

### 正面

- 测试者可以一键保存当前局面与工具调用证据。
- 与工具调用筛选配合后，可以导出更小、更聚焦的调试材料。
- 便于后续人工复盘或回归问题记录。

### 负面

- 当前导出仍受 `limit` 限制，不是全量审计日志归档。
- JSON 调试包可能包含世界状态与审计内容，仍需由使用者自行判断是否适合外发。
