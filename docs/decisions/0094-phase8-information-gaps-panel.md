# ADR-0094: Phase 8 信息缺口面板

## 状态

Accepted

## 背景

右栏已经能展示局势、最近变化、线索、时间线与行动建议，但玩家仍需要在多个面板之间推断“还有哪些信息没摸到”。尤其是未触发剧情线程、未发现观察点、附近顾问是否已请教，分散在剧情、观察、顾问和人物记忆中，复盘成本偏高。

## 决策

在右栏局势分组中新增“信息缺口”面板：

- 汇总尚无线索的剧情线程，展示前几个未触发线程标题与数量。
- 汇总当前场景中高于玩家观察值且尚未发现的观察点，提示可能遗漏细节。
- 汇总当前地点附近尚未请教的顾问，提醒可从顾问面板发起请教。
- 支持信息密度切换，精简模式只显示前两类缺口。

该面板只读取现有 `story_seeds`、`story_progress`、`observations`、实体 `observable_details` 与顾问记忆，不新增规则判断或状态写入。

## 后果

- 玩家可以更快理解当前局势的“未知项”，减少无目标滚动与重复检查。
- 信息展示优化继续保持只读聚合语义，不影响 GM Agent 决策权。
- 由于缺口基于现有素材推导，可能提示“可能遗漏”而非硬性任务清单，符合 AI 原生玩法原则。
