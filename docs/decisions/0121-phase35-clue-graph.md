# ADR-0121: Phase 35 线索关联图可视化

## 背景

Phase 33 增强了调查面板（时间线筛选、跨地域流程图），Phase 34 增强了对话面板。线索面板（`renderCluesPanel`）虽然展示了线索卡片和线程分组，但缺少可视化手段让玩家一眼看出线索之间的关联关系。

## 决策

在线索面板顶部增加 SVG 线索关联图，展示线索、来源实体（NPC）、地点之间的连接关系。

### 图形结构

- **线程背景区域** — 每个线程一个带颜色的半透明矩形，标注线程名称
- **线索节点** — 圆角矩形，按线程着色，显示截断的线索文本（12 字）
- **实体节点** — 圆形（蓝色），显示 NPC ID，通过边连接到引用该 NPC 的线索
- **地点节点** — 胶囊形（绿色），显示地点 ID，通过虚线边连接到发现于该地点的线索

### 布局算法

纯前端计算，无需后端变更：
1. 线程列：左到右排列，每个线程的线索垂直排列
2. 实体列：线索右侧，垂直居中分布
3. 地点列：实体右侧，垂直居中分布
4. 边：从线索节点中心到实体/地点节点中心

### 交互

- 可折叠/展开（默认折叠，点击切换）
- 仅当线索数 ≥ 2 时显示

## 替代方案

1. **D3.js 力导向图** — 更灵活但依赖外部库，增加包体积
2. **Canvas 渲染** — 性能更好但实现复杂度高，SVG 足够当前规模

## 影响

- 前端：新增 `buildClueGraph` / `renderClueGraphSvg` 函数，扩展 `renderCluesPanel`
- 测试：3 个新测试（多线程数据、实体/地点边、空状态）
- 无后端变更，无数据库变更
