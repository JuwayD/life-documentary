# ADR-0029: Phase 8 待处理重点面板

## 状态

Accepted

## 背景

Phase 8 已有局势摘要、右栏导航与导航徽标,但信息仍分散在压力钟、线索、时间线和行动建议等多个面板中。玩家在事件推进后需要滚动多个位置,才能判断“现在最值得注意的是什么”。需要一个更聚焦的展示切片,但不能让前端替 GM Agent 做剧情判断。

## 决策

- 在局势摘要后增加“待处理重点”面板。
- 面板只从现有 `/api/state` 快照聚合展示,最多显示 4 类信息:
  - 压力钟达到 50% 以上时显示最高压力钟。
  - 显示最新记录的线索。
  - 显示最新世界事件。
  - 显示一条下一步建议入口摘要。
- 不新增 API、World 字段、工具或 GM Agent 规则。
- 建议摘要复用前端现有行动建议的优先级:有线索则梳理线索,否则优先请教附近顾问、询问在场者、仔细观察。

## 后果

- 玩家能在右栏顶部快速看到最需要处理的信息,减少滚动和信息过载。
- 面板是展示层聚合,不改变游戏规则或 AI 决策权。
- 若未来行动建议排序变化,待处理重点中的建议摘要需同步维护。
