# ADR-0090: Phase 9 调试快照删除确认

## 状态

Accepted

## 背景

Phase 9 开发者控制台已支持单个和批量删除调试快照。删除会移除本地快照文件,与归档不同不可直接恢复；在列表筛选、批量选择和快速整理测试盘面时,误触删除会丢失复现材料。

## 决策

为测试快照删除操作增加浏览器二次确认：

- 单个删除在调用 `DELETE /api/debug/test-snapshots/{id}` 前弹出确认。
- 批量删除在调用 `POST /api/debug/test-snapshots/bulk-delete` 前弹出确认,并显示所选快照数量。
- 用户取消确认时不发起后端请求,保留列表与选择状态。
- 用户确认后沿用既有删除流程:刷新列表、清理本地详情/差异/加载确认缓存,批量删除后清空选择。

该能力仅约束前端调试控制台操作,不改变快照 API、存档格式或游戏规则判断。

## 后果

- 破坏性调试操作需要显式确认,降低误删测试资产的风险。
- 现有自动化和手工清理流程仍可通过确认后完成。
- 服务端接口保持不变,便于脚本化调试继续直接调用 API。
