# ADR-0058: Phase 8 时间线回合分组

## 状态

Accepted

## 背景

Phase 8 已完成事件时间线筛选、时间线摘要、线索事件联动与最近变化面板。随着单回合内工具事件增多，事件时间线虽然可按类型筛选，但连续事件仍容易混在一起，玩家复盘时需要额外判断哪些事件属于同一轮行动。

## 决策

- 在事件时间线列表中加入轻量分组标题。
- 优先按事件 `turn` 显示“回合 N”；缺少 `turn` 时退回使用事件 `timestamp`；都缺失时显示“未分组事件”。
- 只在相邻事件分组变化时显示标题，避免重复噪音。
- 分组逻辑作用于当前筛选后的可见事件，因此切换类型筛选后只展示剩余事件对应的分组。
- 该改动只读取 `/api/state` 快照中的事件数据，不新增后端接口、不写世界状态。

## 后果

- 玩家可以更快看出连续事件属于哪一轮行动，复盘阅读成本更低。
- 对缺少 `turn` 的历史/工具事件保持兼容，不要求迁移已有事件数据。
- 纯前端展示增强，不影响 GM 决策、工具调用、审计日志或世界状态。
