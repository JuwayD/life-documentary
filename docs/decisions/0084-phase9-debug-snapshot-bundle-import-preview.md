# ADR-0084: Phase 9 调试快照包导入预览

## 状态

Accepted

## 背景

调试快照批量导入已经能把 `mingrpg.test-snapshot-bundle` 追加为本地测试快照列表，但直接选择文件后立即导入，调试者无法先确认包内包含哪些盘面、规模多大，容易误导入不需要的快照。

## 决策

为测试快照包导入增加预览确认流程:

1. 新增 `POST /api/debug/test-snapshots/bulk-import/preview`，复用批量导入的 bundle 格式与快照结构校验。
2. 预览端点只返回包级摘要与每个快照的名称、备注、创建时间和实体 / 地点 / 事件 / flag 数量，不写入本地快照列表。
3. 开发者控制台选择快照包文件后先展示“快照包预览”，列出包内快照摘要。
4. 只有点击“确认导入”后才调用既有批量导入端点。
5. 取消预览会清除待导入包，不改变当前世界和本地快照列表。

## 后果

- 调试者能在导入前确认复现包内容，减少误导入。
- 批量导入仍保持“不自动加载世界状态”的安全边界。
- 后端校验逻辑集中复用，避免预览与实际导入接受不同格式。
