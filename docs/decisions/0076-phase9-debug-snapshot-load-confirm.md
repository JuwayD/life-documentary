# ADR-0076: Phase 9 调试快照加载确认

## 状态

Accepted

## 背景

开发者控制台已经支持测试快照详情预览、差异预览、筛选与排序。但“加载”按钮会直接恢复快照世界状态,在快照较多或名称相近时容易误载,影响当前调试盘面。

## 决策

将开发者控制台测试快照加载改为两步操作:

1. 点击“加载”后先读取快照详情并在当前快照行内展示加载确认摘要。
2. 摘要展示快照名称、玩家、地点、实体/事件/Flag 数量与最近事件。
3. 只有点击“确认加载”才调用现有加载 API 恢复世界。
4. 提供“取消”按钮清除确认摘要。

该切片复用已有 `GET /api/debug/test-snapshots/{id}` 与 `POST /api/debug/test-snapshots/load`,不改变存档格式、不新增后端规则判断。

## 后果

- 降低误载测试快照导致当前调试盘面丢失的风险。
- 快照加载前的关键信息更醒目,与已有详情/差异预览互补。
- 多一步确认增加少量操作成本,但仅影响开发者控制台调试流程。
