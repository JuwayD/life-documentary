# ADR-0071: 调试工具调用输出查看

## 背景

开发者控制台已有工具调用日志筛选能力，但日志卡片只展示工具输入。排查工具副作用时仍需回到审计 JSON 或导出调试包查看输出，不够直接。

## 决策

在开发者控制台的工具调用日志中同时展示工具输入和输出，让开发者在筛选后能直接对照参数与结果。

## 方案

- **前端**：工具调用卡片拆分为“输入 / 输出”两段 JSON 预览
- **样式**：为输入输出标签增加轻量视觉层级
- **后端**：复用现有 `/api/debug/console` 的 `recent_tool_calls.output` 字段，不新增规则判断或端点
- **测试**：覆盖调试 API 返回 output，以及浏览器控制台显示输出内容

## 未选择的方案

- **可折叠 JSON 树**：更适合后续复杂调试，但会增加较多前端状态与交互代码
- **只导出不展示**：保留了完整数据，但不能满足快速定位工具副作用的需求

## 影响

- 无数据迁移
- 无 API 破坏性变更
- 调试控制台工具卡片信息密度略升，但筛选后排查效率提高
