# ADR-0002: 法律与民俗以数据形式存在,非代码

- **状态**: Accepted
- **日期**: 2026-06-06
- **关联**: ADR-0001

## 背景

明朝的法律(大明律)、礼制、江湖规矩、地方民俗等数量庞大,如何让 AI 在判定时引用?

## 决策

**所有法律、民俗、礼制以 YAML 数据形式存储,通过 `query_laws` 工具供 AI 检索。**

```yaml
- id: 大明律.刑律.斗殴.殴制使及本管长官
  category: 斗殴
  era: 明代
  source: 大明律
  text: "凡杖殴本属知府、知州、知县者,杖一百,徒三年"
  applies_to: [violence, against_official, commoner_to_official]
  keywords: [打, 殴, 知府, 知县, 官员]
```

## 关键约束

通过系统提示词强制 AI:
- 涉及法律时**必须**先调用 `query_laws`
- 判定结果**必须**在叙述中引用法条 ID
- 审计日志检查每次判定的 `cited_laws` 字段非空(若动作涉法律)

## 后果

### 正面
- ✓ 法条可由非程序员(策划、历史顾问)维护
- ✓ 人类审核时一眼能看到 AI 引了哪条法
- ✓ 加新法条/民俗 = 加 YAML,不改代码
- ✓ Phase 1+ 可平滑升级为向量检索

### 负面
- ✗ 关键词检索初期可能漏检
  - **对策**: 每条法条加多个 keywords + 同义词;实际验证后补全
- ✗ AI 可能"忘记"调用 query_laws
  - **对策**: 系统提示词反复强调 + 审计告警 + 必要时强制中断重判

## 数据来源

- **大明律**: 公开史料,《大明律集解附例》
- **江湖规矩**: 明清小说常识(三言二拍、水浒等)
- **民俗**: 礼记、地方志选段
- **初期数量**: Phase 0 准备 5-10 条最高频法律
