# ADR-0117: 天气系统

**状态**: 已接受
**日期**: 2026-06-12
**决策者**: 开发团队

## 背景

项目已完成 Phase 30（对话历史面板），拥有 7 个地域、54+ NPC、多 Agent 系统、NPC 对话系统、世界演化系统等。为进一步增强世界沉浸感，需要引入天气系统。

## 决策

实施轻量级天气系统，遵循 AI 原生设计原则：代码只存储数据，所有天气变化决策由 GM Agent 完成。

### 设计原则

1. **数据驱动**: 天气存储在 world flag `weather` 中，包含 condition（类型）、intensity（强度）、text（描述文本）
2. **AI 决策**: GM Agent 通过 `get_weather` 读取天气，通过 `set_weather` 更新天气，何时变化、如何变化完全由 AI 判断
3. **季节默认**: 未设置天气时，根据当前季节返回默认值（春=阴、夏=晴、秋=晴、冬=阴）
4. **叙述融入**: GM 在每回合叙述中自然体现天气，影响 NPC 行为和场景描写

### 天气类型

| 类型 | 英文 | 描述 |
|------|------|------|
| 晴 | clear | 阳光明媚 |
| 阴 | cloudy | 云层遮日 |
| 雨 | rain | 降雨 |
| 暴风雨 | storm | 强风暴雨 |
| 雾 | fog | 雾气弥漫 |
| 雪 | snow | 降雪 |

### 强度等级

| 等级 | 英文 | 描述 |
|------|------|------|
| 轻微 | light | 不太影响活动 |
| 中等 | moderate | 明显影响 |
| 强烈 | heavy | 严重影响 |

### 实现细节

- **`get_weather` 读取工具**: 从 world flag 读取天气，未设置时根据季节返回默认值
- **`set_weather` 写入工具**: 更新天气状态，记录天气变化事件，保留未显式设置的字段
- **GM 系统提示词**: 新增天气系统章节，指引 GM 每回合检查天气、自然融入叙述、根据时间变化天气
- **快照摘要**: 在世界快照中包含天气信息，让 GM 在上下文中看到当前天气
- **前端面板**: 右栏"天气"面板，显示天气图标、类型、强度和描述文本
- **季节初始化**: seed_all 函数中根据季节设置初始天气

## 后果

### 正面

- 增强世界沉浸感，天气影响叙述氛围
- 与现有系统自然集成（NPC 行为、观察检定、支线触发）
- 轻量实现，不增加 API 调用开销
- 完全数据驱动，支持存档/读档

### 风险

- 天气变化频率需要在实际游玩中调优
- GM 可能需要更多上下文来理解天气对不同地域的影响
