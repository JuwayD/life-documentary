# ADR-0061: 设置出生功能

## 状态

已接受

## 背景

当前游戏系统中，玩家角色的初始状态是固定的，无法灵活配置。在测试不同功能（如战斗、交易、技能、社交等）时，需要手动通过多次游戏操作来建立测试条件，效率低下。

需要一个"设置出生"功能，允许在游戏开始前或通过管理界面配置：
- 角色的基础属性（HP、攻击力、防御力等）
- 初始技能等级
- 初始物品和金钱
- 社会背景（出身、身份、声望等）
- 初始位置和场景

## 决策

实现一个 **出生设置系统**，包含以下组件：

### 1. 出生模板 (Birth Templates)
- 预定义的出生配置模板，如"书生"、"商人之子"、"武将后代"、"乞丐"等
- 每个模板包含属性、技能、物品、背景的默认值
- 支持自定义模板

### 2. 属性配置 (Attribute Configuration)
- 基础属性：HP、MP、攻击力、防御力、速度
- 技能等级：文学、武艺、商业、社交、观察等
- 初始物品列表
- 初始金钱

### 3. 背景设定 (Background Settings)
- 出身地：扬州、南京、北京等
- 家庭背景：富商、官宦、平民、没落世家等
- 社会身份：秀才、商人、游侠、乞丐等
- 初始声望值

### 4. 实现方式
- **后端**: 新增 `birth_settings` 工具集
  - `list_birth_templates` - 列出可用模板
  - `get_birth_template` - 获取模板详情
  - `apply_birth_template` - 将模板写入玩家实体并记录出生设置
  - `create_birth_setting` - 创建自定义出生配置
  - `apply_birth_setting` - 应用自定义出生设置到新游戏
- **前端**: 新增出生设置面板
  - 模板选择界面
  - 属性滑块/输入框
  - 背景故事选择
  - 预览功能
- **数据存储**: 出生配置存储为 JSON/YAML 文件

### 5. 与现有系统的集成
- 新游戏开始时可选择出生模板
- 管理界面可快速切换出生配置
- 存档导出/导入时包含出生设置
- 测试模式下可随时重置为指定出生配置

### 6. 已落地的最小应用切片

- `apply_birth_template(world, template_id, entity_id="player")` 将模板的姓名、属性、物品、标签、初始地点与坐标写入玩家实体。
- 应用时清空玩家状态效果，写入 `flags.birth_setting`，并追加 `birth_template_applied` 事件，方便审计与存档追踪。
- Web API 提供 `POST /api/birth/apply` 用于当前世界应用模板。
- Web API 提供 `POST /api/reset?template_id=<id>`，用于测试/新游戏重置时直接套用模板。

### 7. 已落地的前端面板切片

- 顶部工具栏新增“出生设置”入口。
- 出生设置弹层列出模板，展示身份、出身、HP、钱、攻防、观察、技能、物品与起点。
- 点击“以此出生开始”调用 `POST /api/reset?template_id=<id>`，重置世界并按模板刷新状态面板与开场提示。
- 仍不在前端判断模板优劣；模板选择由玩家/测试者决定，代码只展示与应用数据。

### 8. 已落地的管理快速切换切片

- 出生设置弹层顶部新增“当前出生”管理区，读取 `flags.birth_setting.template_id` 标记当前配置。
- 管理区提供一排出生模板快速切换按钮，点击任一模板直接调用 `POST /api/reset?template_id=<id>`。
- 当前模板按钮高亮，切换后复用既有新游戏重置流程刷新聊天、开场、右栏状态与 Pixi 视口。
- 该能力用于测试/调试快速切换角色构建，不新增规则判断或模板优劣排序。

## 后果

### 正面
- 大幅提升测试效率，无需手动建立测试条件
- 支持快速验证不同角色构建下的游戏体验
- 为未来"新游戏+"或多周目系统奠定基础
- 丰富角色扮演体验，增加重玩价值

### 负面
- 增加系统复杂度
- 需要维护出生模板数据
- 可能影响游戏平衡性（需谨慎设计模板数值）
