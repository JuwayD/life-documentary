# ADR-0068: 调试快照差异预览

## 背景

开发者控制台已有测试快照保存/加载功能，但在加载前无法预览快照与当前世界状态的差异。开发者需要"做变更→保存快照→改世界→对比差异"的能力，以验证快照是否包含所需状态、加载后会产生什么变化。

## 决策

为测试快照增加差异比较端点 `/api/debug/test-snapshots/{id}/diff`，在快照列表中增加"差异"按钮，点击后对比当前世界与所选快照。

## 方案

- **后端**：新增 `GET /api/debug/test-snapshots/{snapshot_id}/diff`，返回当前世界 vs 快照的实体/地点/flag 增删改列表以及事件/时间是否改变
- **前端**：快照列表中每个条目增加"差异"按钮，点击后展示差异预览（实体 +N/-N/改N、地点、Flags 分类统计，变化项目列表）
- **测试**：2 个新测试覆盖 diff 正常、404 场景

## 未选择的方案

- **嵌入前端 diff 计算**：数据量较大且逻辑重复，放在后端可复用 World 的 `export_save`，前端只负责展示
- **自动加载后 diff**：已加载到 world 就无法对比，需加载前预览才有意义

## 影响

- 后端新增 1 个 GET 端点，约 50 行 diff 逻辑
- 前端快照列表多一个"差异"按钮，点击折叠差异预览
- 测试新增 5 个断言
- 无数据迁移，向后兼容
