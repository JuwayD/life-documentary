# ADR-0069: 调试快照元数据编辑

## 背景

开发者控制台已有测试快照保存、加载和差异预览能力，但保存后如果名称或备注不准确，只能重新保存一份快照。调试快照承载的是同一份世界状态，改名和补充备注不应影响存档内容。

## 决策

为测试快照增加元数据编辑端点和前端行内编辑区，允许更新快照名称与备注，同时保持 `save` 世界内容不变。

## 方案

- **后端**：新增 `PATCH /api/debug/test-snapshots/{snapshot_id}`，只更新 `name` / `note` / `updated_at`
- **前端**：快照列表每行显示名称与备注输入框，点击“保存备注”后刷新列表
- **测试**：覆盖元数据更新、空名称拒绝、未知快照 404，并验证 diff 摘要不因元数据编辑改变

## 未选择的方案

- **删除后重建快照**：会改变快照 ID 与创建时间，也可能误丢调试上下文
- **只允许编辑备注**：名称同样是定位测试场景的核心元数据

## 影响

- 后端新增 1 个 PATCH 端点，无数据迁移
- 既有快照兼容，缺少 `updated_at` 时仍可正常列出
- 前端调试快照区域增加一行编辑控件
