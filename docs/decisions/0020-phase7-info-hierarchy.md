# ADR-0020: Phase 7 事件/线索信息层级优化

## 状态

Accepted

## 背景

Phase 7 的目标是降低游玩摩擦。事件时间线已经能展示近期事件,下一步需要让玩家更快分辨“最近发生了什么”和“哪些线索属于哪个线程”。

## 决策

- 继续复用 `/api/state` 中已有 `events` 与 `flags.story_progress`,不新增后端 API 或规则判断。
- 时间线事件按既有字段推导展示类型标签,并标注最新事件。
- 线索面板增加总览行,主线优先,其他线程按线索数量与名称排序。
- 每个线程内新线索在上,最新线索显示“新近”标记。

## 后果

- 玩家能更快扫描右栏信息,减少在多个面板之间回忆上下文的成本。
- 代码只做展示分层,不判断线索重要性或剧情走向。
- 后续“行动建议入口”可以基于这些面板旁路提供入口,但仍应由 LLM 生成建议。
