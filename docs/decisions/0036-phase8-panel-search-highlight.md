# ADR-0036: Phase 8 搜索命中高亮

## 状态

Accepted

## 背景

ADR-0035 已在右栏加入面板搜索筛选。搜索能减少需要浏览的面板数量，但当匹配来自面板正文而非标题时，玩家仍需要在可见面板内继续扫读才能找到命中位置。

## 决策

- 在右栏搜索筛选保留原有显示/隐藏逻辑的基础上，高亮可见面板中的匹配文字。
- 高亮仅作用于当前 DOM 文本节点：
  - 输入关键词后，为匹配片段包裹 `mark.panel-search-highlight`。
  - 清空搜索或重新应用搜索前，移除旧高亮并恢复普通文本节点。
  - 状态刷新后由现有 `renderSidePanel` 重新应用当前搜索词。
- 不新增后端接口、世界状态、工具层或 GM Agent 规则。

## 后果

- 玩家不仅知道哪些面板匹配，还能快速看到面板内具体命中位置。
- 实现仍完全停留在前端展示层，风险低，可通过 Playwright E2E 独立验收。
- DOM 文本节点会被局部替换；当前高亮只包裹文本，不改交互按钮属性或业务数据。
