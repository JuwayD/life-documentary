# ADR-0024: Phase 8 渲染签名缓存

## 状态

Accepted

## 背景

Phase 8 开始关注实时渲染性能。当前 Web 前端在收到 turn 完成状态、导入存档、重置或重复调用 `renderScene` 时,即使玩家位置、当前地点、附近实体和 HP 等渲染输入没有变化,仍会进入 Pixi 更新流程。

## 决策

- 在前端为 Pixi 视口增加渲染签名缓存。
- 签名只包含影响当前视口的事实:玩家当前位置、地点尺寸/出口/标签、当前地点实体的 id/name/type/location/pos/hp/max_hp/tags。
- 当签名未变化时直接跳过 Pixi 重绘和实体更新。
- 保持侧栏、对话、审计等非 Pixi UI 原有刷新路径不变。
- 不新增游戏规则判断,不改变 World、工具层或 GM Agent 行为。

## 后果

- 重复状态推送或重复 `renderScene` 调用不会触发无意义 Pixi 重绘。
- 实体移动、HP 变化、进出当前地点、地点出口/尺寸/标签变化仍会触发正常渲染。
- 当前优化是浏览器端最小切片;后续 Phase 8 可继续做节流、脏区域更新或信息展示优化。
