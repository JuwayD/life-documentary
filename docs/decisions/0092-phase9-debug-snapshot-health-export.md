# ADR-0092: Phase 9 调试快照健康报告导出

## 状态

Accepted

## 背景

调试快照健康校验已经能在开发者控制台发现损坏 JSON、重复名称、旧格式、缺少玩家或地点等问题。但报告只停留在当前浏览器界面,不便于将坏档诊断结果发给协作者、归档到问题记录或附加到回归说明中。

## 决策

新增只读的调试快照健康报告导出能力：

- 后端提供 `GET /api/debug/test-snapshots/health/export`。
- 导出内容复用现有健康校验结果,并补充 `format: mingrpg.test-snapshot-health`、`version: 1`、`exported_at`。
- 前端开发者控制台新增“导出校验报告”按钮,下载 JSON 文件。
- 导出时同步刷新控制台内的健康报告展示。

该能力只读取快照目录并生成诊断 JSON,不修复、不删除、不加载快照,不改变当前世界状态。

## 后果

- 调试者可以把快照坏档、重复命名和结构缺口作为独立 JSON 报告分享。
- 后续排查无需复现完整浏览器环境即可查看诊断摘要。
- 新增端点复用既有校验逻辑,避免健康校验和导出结果不一致。
