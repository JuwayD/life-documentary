# ADR-0031: Phase 8 事件时间线筛选

## 状态

Accepted

## 背景

Phase 8 已增加局势摘要、待处理重点和关系动态面板,右栏信息密度继续上升。事件时间线虽然能展示最近事件,但在复盘时剧情、社交、交易、战斗和其他事件混在一起,玩家需要滚动和人工筛选。需要补一个最小展示层切片,降低复盘噪音,但不改变事件存储和 GM Agent 决策。

## 决策

- 在“事件时间线”面板顶部增加类型筛选按钮:全部、剧情、社交、战斗、交易、其他。
- 继续使用前端基于事件字段的轻量分类,不新增后端接口、事件 schema 或工具。
- 筛选按钮显示当前最近事件中的类型数量,无事件的类型置灰。
- 当前筛选类型若在新状态中不存在,自动回到“全部”,避免空筛选困住玩家。

## 后果

- 玩家可快速只看线索/剧情事件或社交互动事件,提升复盘效率。
- 实现保持为展示层聚合,不影响审计日志、世界状态或 AI 规则判断。
- 后续若事件类型字段更稳定,筛选逻辑可从关键词检测切换为直接读取标准类型。
