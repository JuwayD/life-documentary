# ADR-0086: Phase 9 调试快照复制

## 状态

Accepted

## 背景

Phase 9 已支持测试快照保存、标签、筛选、排序、导入导出与批量操作。开发者在调试时常需要基于一个稳定盘面派生多个相近回归场景；如果只能重新保存当前世界，容易把后续临时操作混入快照。

## 决策

为测试快照新增复制能力：

- 后端新增 `POST /api/debug/test-snapshots/{snapshot_id}/duplicate`。
- 默认复制为“原名 副本”，也允许提交新名称、备注、标签。
- 新快照使用新的 id、创建/更新时间，并记录 `duplicated_from`。
- 副本保留源快照的 `save` 存档内容，不读取或覆盖当前世界状态。
- 前端开发者控制台在每个快照行新增“复制”按钮，复制后刷新列表并提示。

复制只操作测试快照元数据与存档副本，不进入世界状态，也不参与任何游戏规则判断。

## 后果

- 可从稳定盘面快速派生多个命名不同、标签不同的回归快照。
- 当前世界中后续临时调试动作不会污染复制出的快照内容。
- 复制出的快照仍可继续编辑、导出、批量选择和删除。
