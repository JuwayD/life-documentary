# ADR-0089: Phase 9 调试快照批量归档

## 状态

Accepted

## 背景

Phase 9 开发者控制台已支持调试快照归档,但整理多份临时快照时仍需逐条点击。批量删除虽可清理列表,但会丢失复盘材料；批量归档更适合非破坏性降噪。

## 决策

为测试快照新增批量归档能力：

- 新增 `POST /api/debug/test-snapshots/bulk-archive`。
- 请求体包含 `snapshot_ids` 与 `archived` 布尔值,支持批量归档和批量取消归档。
- 服务端复用快照 id 去重和存在性校验,逐个更新 `archived` 与 `updated_at`。
- 响应返回更新后的快照元数据和列表汇总。
- 开发者控制台批量操作区新增“归档所选”和“取消归档”按钮。
- 批量归档后清空选择并刷新列表；默认隐藏已归档快照,可通过“显示归档”恢复查看。

该能力只修改调试快照元数据,不读取、不导入、不覆盖当前世界状态,也不参与游戏规则判断。

## 后果

- 多份临时测试盘面可以快速从默认列表降噪隐藏。
- 取消归档同样批量化,方便恢复误归档快照。
- 相比批量删除,该操作保留快照文件和导出复盘能力,更适合日常调试整理。
