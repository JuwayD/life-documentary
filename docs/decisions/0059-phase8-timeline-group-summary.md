# ADR-0059: Phase 8 时间线组内类型摘要

## 状态

Accepted

## 背景

Phase 8 已为事件时间线加入筛选、摘要与按回合分组。分组能说明“这些事件属于同一回合”，但玩家仍需要逐条阅读才能知道该回合主要发生了剧情推进、社交互动还是交易/战斗。

## 决策

- 在事件时间线分组标题右侧增加组内摘要。
- 摘要展示当前可见分组内的事件总数与类型构成，例如“2件 · 剧情2”或“3件 · 社交1 / 剧情2”。
- 组内摘要基于当前筛选后的事件列表重新计算；切换类型筛选后，摘要只反映仍可见的事件。
- 复用既有 `detectTimelineType` 与类型标签，不引入后端规则或新接口。
- 该改动只增强前端展示，不写入世界状态。

## 后果

- 玩家扫读时间线时能更快判断每个回合的事件密度与主要性质。
- 类型筛选与回合分组保持一致，不会显示已被筛掉的事件统计。
- 纯 UI 增强，不影响 GM Agent、工具调用、审计日志或存档格式。
