# 人生纪实 · 明朝篇

一款以明代万历年间为背景的 AI 原生文字角色扮演游戏。

## 核心理念

> **代码是手脚,AI 是大脑。**
> 代码只读写数据、提供工具,不做任何游戏规则判断。所有决策、推理、叙述由 LLM 完成。

详见 [`docs/00-vision.md`](docs/00-vision.md) 和 [`docs/01-principles.md`](docs/01-principles.md)。

## 快速开始

```bash
python3 -m venv .venv
.venv/bin/pip install -e ".[dev]"
.venv/bin/playwright install chromium     # 仅 E2E 测试需要

# 跑全部测试 (752+ 单元/E2E + 4 LLM 集成 skip,全过)
.venv/bin/pytest

# 跑 live LLM 集成测试 (~4 分钟)
MINGRPG_RUN_LIVE=1 .venv/bin/pytest tests/integration/

# === 开始游戏 ===

# (A) Web UI(推荐)
./run_web.sh                              # 自动开浏览器
# 或: .venv/bin/python -m mingrpg.web.server

# (B) CLI 文字版
.venv/bin/python -m mingrpg.cli
```

打开 http://127.0.0.1:8765 即可游玩。

## 默认场景

万历十年,扬州府衙大堂。你是一名落第书生,手持状纸求见王知府。

试着输入:
- "我向王知府深深一揖,呈上状纸"
- "我冲上去给王知府一拳!"
- "我转身向师爷使个眼色"
- "我大喊冤枉,跪在地上不肯起来"

GM Agent 会:
- 读取你和周围 NPC 的状态
- 检索相关法条(大明律)
- 必要时掷骰判定
- 修改世界状态(给 NPC 加记忆、改你的位置等)
- 生成符合明代官场风格的叙述

## 项目结构

```
人生纪实/
├── docs/                 项目文档 + 架构决策(ADR)+ 会话日志
├── src/mingrpg/
│   ├── core/             World 状态层 (SQLite)
│   ├── tools/            工具层 (read / write / util)
│   ├── llm/              GM Agent + 工具注册表
│   ├── audit/            审计日志 (JSONL)
│   ├── scenarios/        场景种子数据
│   ├── web/              FastAPI 服务 + 静态前端
│   │   └── static/       index.html / style.css / app.js
│   └── cli.py            命令行入口
├── data/laws/            大明律 YAML 数据
├── tests/                单元 + 集成 + Playwright E2E
└── run_web.sh            一键启动 web
```
